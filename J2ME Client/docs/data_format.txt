Транспорт предполагается быть бинарным.

Команда состоит из следующих частей:

 Номер параметра	Размер параметра, байт 	 Название	 Примечания
 1	 4	 Размер команды	 
 2 	 16	Тикет	 GUID -- идентификатор соединения
 3	 1	 Номер компонента	 Сервер имеет перечисление из имеющихся компонент. Например 1-это компонент аутентификации, 2 - это компонент игровой логики, 3 - компонент чата и т.д. 
 4	 1	 Номер системы	 Для компонента игровой логики это 1 - это система большой карты, 2 - система боя, 3 - торговая система и т.д.
 5	 1	 Номер команды	 Для системы боя это будут команды: 1 - сделать ход юнитом, 2 - изменить режим атаки юнита
 6...	 	 	 Различные параметры, специфичные для данной команды

Особое внимание следует уделить передаче строк и коллекций.

В этом случае передается размер, затем идет информация, то есть строка '123' в команде будет выглядеть как:

0x00 0x03 0x31 0x32 0x33, поскольку первые два байта используются под размер строки (коллекции).
